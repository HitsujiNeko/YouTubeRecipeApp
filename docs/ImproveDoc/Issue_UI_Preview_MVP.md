# 課題



## HOME画面
- YouTubeURLを入力しやすいように、Youtubeへ遷移する導線を設ける。
- 「最近のレシピ」は、ユーザのマンネリ化を引き起こす可能性があるため、おすすめのレシピなどの表示も検討
- 最近のレシピは、文字列情報のみ→レシピカードの検討

## RecipeDetail画面
- COOKMODEに遷移する導線があるか確認（最優先）
- 画像がなく、文字情報のみのため、淡泊な印象を受ける。
    →サムネイル画像を表示する
- 材料の鶏むね肉が1枚と表記されているが、ユーザによって大きさが異なるため、グラム数で表記することを検討する。
- confidence/coverageの数値がユーザにとってわかりづらい可能性があるため、UI上での表現方法を工夫する必要がある。
    →例: confidenceは星の数、coverageはバーの長さなど視覚的にわかりやすい表現を検討する。
- confidence/coverageが低い場合、なぜ低いのかがわからない。


## COOKMODE画面
- YouTube動画の埋め込みを行い、手順や材料など文字情報と同時に動画も確認できるようにする。（優先度高）
-

## 全体
- コンポーネントとしてレシピカードを実装する。
これにより、視覚的にレシピの内容を把握しやすくなり、ユーザの理解が深まる。また、レシピカードコンポーネントを再利用することで、開発効率が向上する。


また、別アプリで一度、レシピカードコンポーネントを実装したことがある。参考になるようにdocs\ImproveDoc\RecipeCard.tsxにコードを記載する。

- 追加画面として、「レシピ一覧画面」を実装する
これにより、ユーザは過去に生成したレシピを一覧で確認できるようになる。また、食材やカテゴリ、タグ、YouTubeチャンネルでフィルタリングすることができる機能を設ければ、以下の効果が期待できる。
    1. ユーザは家にある食材でつくることができるレシピを簡単に見つけることができる。
    2. ユーザは自分の好みに合ったレシピを見つけることができる。
    3. レシピ数が蓄積しても、ユーザが目的のレシピを見つけやすくなる。
