# 03 課題→解決→優先順位 v3（P0を確定）

## 3.1 課題（ユーザーの痛み）
C1: 調理中、動画を止める/戻す回数が多く、初心者ほどつらい
C2: 概要欄がノイズ（リンク/道具/PR）で材料が拾えない
C3: 作る前に栄養が分からず、健康目的の意思決定ができない
C4: 栄養管理アプリへの手入力が面倒で続かない

## 3.2 解決（アプリ価値）
S1: URL→レシピカード（材料/手順）で調理中の摩擦を減らす
S2: ノイズ除去 + チャンネル特化で抽出精度を上げる
S3: 推定栄養（PFC/kcal/食塩）を生成し、作る前に選べる
S4: ちょい編集（候補選択+g入力）で、信頼性を上げるが“入力地獄”にはしない

## 3.3 P0（MVP必須）
- URLインポート（YouTube meta取得）
- レシピカード生成（材料/手順 JSON）
- クックモード（チェック・1ステップ・タイマー）
- 栄養推定（kcal/PFC/食塩）
- confidence/coverage表示（未確定材料リスト）
- ちょい編集UI（材料候補選択、g入力）
- 保存/検索（最低限）
- 共有（カードURL + 元動画リンク）※グロースの核なのでP0推奨

## 3.4 P1（成長）
- 自動タグ（高P/低脂質/減塩/時短等）
- 健康フィルタ検索（P/100kcalや脂質上限など）
- チャンネル専用パーサ（上位3）
- 章ジャンプ（概要欄にタイムスタンプがある場合）

## 3.5 P2（マネタイズ強化）
- 翻訳（海外レシピ→日本語カード+栄養）
- 目標設定（減量/増量）と推薦ランキング
- 詳細栄養（ビタミン/ミネラル等）
